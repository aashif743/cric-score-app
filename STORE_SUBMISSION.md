## Common Rejection Reasons to Avoid

### iOS
1. **Incomplete Information** - Ensure all metadata is filled
2. **Broken Links** - Privacy Policy URL must be accessible
3. **Crashes** - Test on multiple devices
4. **Login Issues** - Provide test account for review
5. **Placeholder Content** - Remove any "Lorem ipsum"
6. **Misleading Metadata** - Description matches app features
7. **Background Location** - Not using, so not an issue

### Android
1. **Privacy Policy Missing** - Must be valid URL
2. **Data Safety Inaccurate** - Be honest about data collection
3. **Target API Too Low** - Ensure API 33+ target
4. **Crashes on Launch** - Test thoroughly
5. **Permissions Not Justified** - Only request needed permissions
6. **Deceptive Behavior** - App does what it says
